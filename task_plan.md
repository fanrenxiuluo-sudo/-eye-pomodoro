# EyeTimer v0.3.0 — 升级计划

## 问题诊断

### 问题 1：UI 太简陋
- **现状**: TimerPage 只有一个 SVG 圆环 + 时间 + 三个按钮，没有间距呼吸感
- **现状**: SettingsPage 是纯表单排列，无分组视觉层次
- **现状**: 全局 CSS 只有基础变量 + 无渐变/微动效
- **需要**: 现代化设计，渐变色，玻璃拟态，更丰富的微动效，信息密度更高

### 问题 2：检查更新必须去浏览器
- **现状**: `openReleasePage()` 直接 `shell.openExternal()`，用户被丢到浏览器
- **现状**: `electron-updater` 的 `autoUpdater` 已经接入，但配置可能未正确指向 GitHub Releases asset
- **需要**: 去掉"在浏览器中查看"按钮，完善应用内检查→下载→安装的完整流程

### 问题 3：专注时间到提示太弱
- **现状**: 提示音是合成正弦波，音量可能不够
- **现状**: Windows 系统通知容易被忽略（静音、勿扰模式）
- **现状**: 窗口仅 `show()` + `focus()`，但无视觉冲击
- **需要**: 
  - 大幅提高提示音辨识度（重复次数、音量、频率变化）
  - 全屏闪烁/脉冲特效（主窗口 + 所有显示器）
  - 任务栏闪烁 (flashFrame)
  - 更醒目的系统通知

### 问题 4：休息时段缺少护眼指导
- **现状**: overlay 的 TIPS 只有 8 条简短文案，无科学依据
- **需要**: 基于权威护眼研究（20-20-20 法则、眼保健操、远近交替对焦等）的完整休息指导系统
  - 短休息（5分钟）：快速眼部放松动作
  - 长休息（15-30分钟）：完整护眼操 + 身体活动
  - 每条提示附带来源说明
